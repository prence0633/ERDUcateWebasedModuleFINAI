Introduction- Table Structures & Keys

In this lesson, students learn how Entities and Attributes are stored in Tables. Frankie and John explore how Rows represent entities (records) and Columns represent attributes (fields). They also see how Primary Keys uniquely identify each row and how Foreign Keys connect tables to maintain relationships. By organizing data this way, students understand how tables relate to each other and how real-world scenarios can be modeled efficiently in a database.

Summary ‚Äî Tables & Keys

Table: Stores Entities (rows) and Attributes (columns).

Primary Key: Unique identifier for each row (e.g., StudentID).

Foreign Key: A column that links one table to another.

Glossary ‚Äî Key Terms

Table
A collection of rows (records) and columns (attributes).
Row
A single record in a table (an entity instance).
Column
An attribute or field that stores one type of data for all rows.
Primary Key
A unique identifier for each row.
Foreign Key
A column referencing a Primary Key in another table, used to create relationships.
